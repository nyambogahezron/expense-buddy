# Issues

This document outlines the guidelines for reporting issues in the Expense Buddy project.

## Reporting Bugs

When reporting a bug, please include:

- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots or error messages (if applicable)
- Your environment (OS, browser, version of the app)

## Feature Requests

For feature requests, please provide:

- A clear description of the feature
- Why you think it's needed
- Any mockups or examples (if available)

## General Guidelines

- Check if the issue has already been reported
- Use a clear and descriptive title
- Provide as much detail as possible
- Be respectful and constructive

## Labels

Issues will be labeled according to their type and priority:

- `bug`: Something isn't working
- `enhancement`: New feature or request
- `documentation`: Improvements or additions to documentation
- `question`: Further information is requested
- `high-priority`: Needs immediate attention
- `low-priority`: Can be addressed later
